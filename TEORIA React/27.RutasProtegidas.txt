Rutas Protegidas
Hemos logrado ocultar el enlace a la página de "Alta de Productos". Sin embargo, nos encontramos con dos problemas:
1. Si un usuario conoce la URL (/alta) y la escribe directamente en el navegador, aún podría acceder. Esto es un gran fallo 
de seguridad.
2. Peor aún, si un usuario se loguea en nuestro sitio, sin importar si es admin o no, va a tener acceso a la sección de Gestion.
Una Ruta Protegida es un mecanismo que nos permite controlar el acceso a una ruta de nuestra aplicación basándonos en 
ciertas condiciones, como por ejemplo, si el usuario está autenticado, el rol, incluso el lugar donde se loguea.

Creación del componente ProtectedRoute
Crearemos un componente genérico que se encargará de esta lógica de protección. Este componente "envolverá" a cualquier 
otra ruta que deseemos proteger.

//src/componentes/ProtectedRoute/PotectedRoute.jsx
import React from 'react';
import { useAuth } from '../../context/AuthContext';
import { Navigate } from 'react-router-dom';
const ProtectedRoute = ({ children }) => {
const { user, loading } = useAuth();
// Mientras se verifica el estado de autenticación, mostramos un mensaje de carga.
src/componentes/ProtectedRoute/PotectedRoute.jsx
// Esto es crucial para no redirigir al login prematuramente en una
recarga de página.
if (loading) {
return <div>Cargando...</div>;
}
// Si no hay un usuario autenticado O si su rol no está incluido en la
lista de roles permitidos, redirigimos al login.
if (!user || (rolesPermitidos && !rolesPermitidos.includes(user.rol))) {
return <Navigate to="/login" />;
}
}
// Si hay un usuario, renderizamos el componente hijo que está siendo
protegido.
return <>{children}</>;
};
export default ProtectedRoute;

Análisis del Código:
● El componente recibe children, que es el componente que queremos proteger (por ejemplo, la página de Alta).
● Utiliza nuestro hook useAuth para acceder a user y al estado loading.
● Si aún está loading, no hace nada más que esperar.
● Si ya terminó de cargar y no hay user, utiliza el componente <Navigate> de react-router-dom para forzar una redirección 
a la ruta /login.
● Solo si terminó de cargar y user existe, renderiza los children.

Implementación en el Router
Ahora, aplicar esta protección es sumamente sencillo. En nuestro archivo de rutas (generalmente App.jsx), envolvemos el 
element de la ruta que queremos proteger con nuestro nuevo componente ProtectedRoute.

//src/App.jsx
import ProtectedRoute from './componentes/ProtectedRoute/ProtectedRoute';
function App() {
return (
<Routes> {/*Otras rutas*/}
<Route
path="/alta"
element={
<ProtectedRoute rolesPermitidos={['admin']}>
<Gestion />
</ProtectedRoute>
}
/>
</Route>
</Routes>

El componente ahora recibe una prop rolesPermitidos, que será un array con los strings de los roles que pueden acceder a esa 
ruta en particular (ej: ['admin']). Si no estás logueado O si, estando logueado, tu rol no está en la lista de roles 
permitidos se te redirige al login.
