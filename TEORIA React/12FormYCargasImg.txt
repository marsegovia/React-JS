Otros eventos:
La clase 4 vimos el evento onClick que nos permitía accionar frente a al clic del usuario en algún elemento: en 
nuestro caso en un boton. Con las mismas reglas que usamos onClick, vamos a recordar estos dos nuevos eventos que 
nos van a permitir interaccionar con el formulario: onChange y onSubmit.

Evento onChange
Captura cambios en tiempo real en elementos de formulario (inputs, selects, textareas). Por ejemplo:

<input
value={nombre}
onChange={(e) => setNombre(e.target.value)}
/>

Descripción:
1. Usuario escribe → Dispara evento onChange
2. e.target.value → Obtiene el valor actual del input
3. setNombre() → Actualiza el estado de React
4. Re-render → El input muestra el nuevo valor
Características:
✅ Se ejecuta con cada tecla presionada
✅ Sincroniza el estado con lo que el usuario ve
✅ Esencial para formularios controlados

Evento onSubmit
Maneja el envío del formulario cuando el usuario hace submit. Por ejemplo:

<form onSubmit={(e) => {
e.preventDefault();
console.log('Formulario enviado!');}}>

1. Usuario envía form (Enter o click en botón)
2. e.preventDefault() → Evita recargar toda la página
3. Ejecuta lógica → Validación, enviar datos, etc. (console.log en nuestro caso)
Características:
✅ Se ejecuta una vez al enviar el form
✅ Previene la recarga de la página, comportamiento que viene por default.
✅ Ideal para procesar datos finales

Creando nuestro formulario.
Vamos a usar estos eventos en un formulario real, que nos permita capturar la información y guardarla. Para
eso dividimos la creación del formulario en 3 pasos:
1. Creamos el formulario
2. Lo hacemos parte de nuestro patron Contenedor/Presentacional
3. Lo integramos a nuestra aplicación

Paso 1: Un formulario presentacional simple
Recordemos lo que vimos en la clase 4: los componentes presentacionales solo se preocupan por cómo se ven las cosas.
No tienen lógica compleja. Vamos a empezar creando un formulario simple que solo renderiza los campos que necesitamos.

Dándole agilidad
Modifiquemos nuestro componente para que reciba 3 props: datosForm, manejarCambio
y manejarEnvio. Estos nos van a servir para:
● datosForm: Un objeto que contiene los valores actuales de todos los campos del formulario (nombre, precio, 
stock, etc.). Lo usaremos para poblar el atributo value de cada input.
● manejarCambio: Una función que se ejecutará cada vez que el usuario escriba en un input. La conectaremos al 
evento onChange.
● manejarEnvio: Una función que se ejecutará cuando el usuario envíe el formulario. La conectaremos al evento 
onSubmit de la etiqueta <form>.
Al formulario le agregamos la opción de onSubmit con su prop correspondiente. También modificamos los input para 
que podamos manejar los datos con sus props.

// En src/componentes/FormularioProducto/FormularioProducto.jsx
import React from 'react';
// Recibe los datos y las funciones por props.
export function FormularioProducto({ datosForm, manejarCambio, manejarEnvio })
{
const formStyle = { // mismo diseño };
return (
<form style={formStyle} onSubmit={manejarEnvio}>
<h3>Agregar Nuevo Producto</h3>

<div>
    <label>Nombre del Producto:</label>
    <input type="text" placeholder="Ej: Teclado Mecánico" name="nombre" value={datosForm.nombre} onChange={manejarCambio}/>
</div>

<div>
    <label>Precio: $</label>
    <input type="number" placeholder="Ej: 95" name="precio" // Atributo clave value={datosForm.precio} onChange={manejarCambio}/>
</div>
// Continuamos con los otros 2 atributos del formulario.
<button type="submit">Guardar Producto</button>
</form>
);
}

Paso 2. Creamos el componente contenedor del Formulario
Una vez que tenemos nuestro componente presentacional listo, vamos a generar su contenedor. Para eso creamos un 
nuevo componente llamado FormularioContainer, donde importamos useState y el componente FormularioProducto.

Este componente será el cerebro de la operación.
1. Manejo del Estado con useState: Lo primero que hacemos es utilizar useState para crear una variable de estado 
llamada datosForm, donde agrupamos todos los campos en un único objeto(nombre, precio, etc.),
2. Lógica de Actualización: Creamos una única función, manejarCambio, que permite actualizar cualquier campo del
formulario, haciendo que cada vez que el usuario escribe, se dispare el evento onChange.
Recibe el objeto evento y extrae name (el atributo name que le dimos a cada input, como "nombre" o "precio") y 
value (el texto que el usuario está escribiendo).

const manejarCambio = (evento) => {
const { name, value } = evento.target;
setDatosForm({
...datosForm, [name]: value
});
};

3. Lógica de Envío: manejarEnvio
Esta función se ejecuta cuando el usuario presiona el botón "Guardar producto". Primero llama a 
evento.preventDefault() para evitar el comportamiento por defecto de recargar la página entera al enviarse.
Después centralizamos todo lo que debe ocurrir con los datos ya completos. Por ahora, simplemente los mostramos en 
la consola, pero próximamente desde aquí haríamos la llamada a una API para guardar el producto en una base de datos.

const manejarEnvio = (evento) => {
evento.preventDefault();
console.log('Enviando los siguientes datos a la API:', datosForm);
};

4. Return
Este no renderiza HTML directamente. Su única función es renderizar al componente presentacional FormularioProducto 
y pasarle, a través de props, todo lo que necesita para funcionar.

// En src/contenedores/FormularioContainer/FormularioContainer.jsx
import React, { useState } from 'react';
import { FormularioProducto } from '../FormularioProducto/FormularioProducto';
export function FormularioContainer() {
const [datosForm, setDatosForm] = useState({
nombre: '',
precio: '',
stock: '',
urlImagen: ''
});
const manejarCambio = (evento) => {
const { name, value } = evento.target;
setDatosForm({
...datosForm,
[name]: value
});
};

const manejarEnvio = (evento) => {
evento.preventDefault();
console.log('Enviando los siguientes datos a la API:', datosForm);
};
return (
<FormularioProducto
datosForm={datosForm}
manejarCambio={manejarCambio}
manejarEnvio={manejarEnvio}
/>
);
}
