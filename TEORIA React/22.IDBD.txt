Acceso al detalle de un producto con ID de Firebase

Ahora que cada producto tiene un ID único en Firebase, podemos implementar esta funcionalidad por completo.
Tenemos que hacer lo mismo que hicimos anteriormente con el componente Productos, pero en este caso, vamos a consultar a Firebase.
Para esto, necesitamos tres herramientas clave:
● <Link> de React Router: Para que cada producto de nuestra lista nos lleve a su página de detalle.
● Hook useParams: Para capturar el itemId desde la URL.
● Funciones doc y getDoc de Firebase: Para solicitar un único documento por su ID.

Paso 1: Enlazar desde la lista de productos y en App.jsx
En el componente que renderiza la lista de productos nacionales, modificaremos el mapeo para que cada producto sea un enlace.

    {/* En /src/componentes/ProductosNacionales/ProductosNacionales.jsx */}
    
    <Link to={`/productos-nacionales/${prod.id}`}>Ver detalle</Link>
    {/* En /src/App.jsx */}
    <Route path="/productos-nacionales/:id" element={<ProductosNacionalesDetalle/>} />

Paso 2: Crear el contenedor del detalle
Este nuevo componente se encargará de buscar y mostrar la información del producto seleccionado, será muy parecido al componente 
ProductoDetalle que hicimos anteriormente, por lo que podrás reutilizar muchas cosas de este.

Análisis del código:
Usamos el hook useParams para desestructurar y tomar su Id y el hook useEffect para la petición de datos. En este caso, el array 
de dependencias [id] asegura que el hook se vuelva a ejecutar si el ID en la URL cambia.
1. doc(): A diferencia de collection(), doc() crea una referencia a un documento específico. Requiere tres argumentos: 
la configuración de la base de datos (db), el nombre de la colección, y el ID del documento que buscamos.
2. getDoc(): Es el análogo a getDocs(), pero para un solo documento. Devuelve una promesa con la respuesta.
3. resp.exists(): Es crucial verificar si el documento fue encontrado antes de intentar acceder a sus datos con resp.data().
