Introducción a la optimización de estilos

uso de componentes de React-Bootstrap y la estilización granular con styled-components.

Ajustes de responsividad con React-Bootstrap
Cuando trabajamos con React, en lugar de escribir directamente clases de Bootstrap en className, podemos usar componentes ya 
preparados por React-Bootstrap. Esto nos da varias ventajas:
● Código más limpio y fácil de leer.
● Evitamos errores de sintaxis en las clases de CSS.
● Usamos un enfoque declarativo: describimos el diseño con componentes en lugar de reglas CSS.

Instalación de Dependencias:
Para comenzar, debemos instalar tanto react-bootstrap como la librería bootstrap en nuestro proyecto.

npm install react-bootstrap bootstrap

Luego, es fundamental importar la hoja de estilos de Bootstrap en el punto de entrada de nuestra aplicación, 
típicamente index.js o App.jsx

// En /src/index.js o /src/App.jsx
import 'bootstrap/dist/css/bootstrap.min.css';

⚠️ Si olvidamos este paso, los componentes se verán sin estilos.

Ejemplo aplicado a productos
En vez de usar <div> con clases, usamos componentes de Bootstrap como:
● Container: agrupa todo el contenido y aplica márgenes automáticos.
● Row: crea filas donde se ubican las columnas.
● Col: divide en columnas y se adapta según el tamaño de pantalla.
● Card: muestra cada producto con imagen, título y botón.

// src/componentes/productosNacionales/ProductosNacionales.jsx
// Otras importaciones
    import { Container, Row, Col, Card, Button } from 'react-bootstrap';
    const ProductosNacionales = () => {

    // Resto del código
    }, []);

    return (
        <Container className="mt-4">
        <Row>
        {productos.map(prod => (
        <Col key={prod.id} xs={12} md={6} lg={4} className="mb-4">
        <Card className="h-100">

        <Card.Body className="d-flex flex-column">

    </Row>
    </Container>

export default ProductosNacionales;

Explicación del código:
En lugar de usar <div> con clases de Bootstrap, en React-Bootstrap trabajamos con componentes ya preparados como <Container>, 
<Row> y <Col>.
La responsividad se logra gracias a las props xs, md y lg dentro de <Col>. Cada una representa un breakpoint (punto de quiebre)
según el ancho de la pantalla:
    xs={12} → en móviles, cada tarjeta ocupa todo el ancho (una sola columna).
    md={6} → en tablets, cada tarjeta ocupa la mitad (dos columnas por fila).
    lg={4} → en escritorios, cada tarjeta ocupa un tercio (tres columnas por fila).
Esto nos permite diseñar de forma declarativa e intuitiva, sin tener que escribir reglas CSS extra.

Puntos importantes para entender React-Bootstrap
1. Grid System: usa el sistema de 12 columnas de Bootstrap, pero expresado como props en los componentes <Col>.
2. Breakpoints: los más comunes son xs, sm, md, lg, xl (de menor a mayor tamaño de pantalla).
3. Estilos predefinidos: no necesitamos crear CSS desde cero; los componentes ya aplican estilos modernos y responsivos.
4. Compatibilidad: todo lo que funciona en Bootstrap (clases utilitarias como mt-4, d-flex, etc.) también lo podemos usar 
junto a React-Bootstrap.

Estilización a nivel de componente con styled-components
Cuando Bootstrap no alcanza o queremos un diseño más personalizado y único, podemos usar la librería styled-components.
Esta nos permite crear componentes de React con estilos encapsulados (no afectan a otros elementos) y totalmente reutilizables.

Instalación de Dependencias:

npm install styled-components

Ejemplo: Creando un botón personalizado Queremos reemplazar los botones genéricos de Bootstrap por unos con estilo propio. 
Para eso vamos a personalizar los botones de "Editar" y "Eliminar" en Gestion.jsx.

import styled from "styled-components";
// 1. Definimos un componente base para los botones
const BotonAccion = styled.button`
background-color: transparent;
border: 1px solid #ccc;
border-radius: 5px;
padding: 5px 10px;
cursor: pointer;
margin-left: 8px;
transition: all 0.2s ease;
&:hover {
transform: translateY(-2px);
box-shadow: 0 2px 5px rgba(0,0,0,0.1);
}
`;
// 2. Extendemos el estilo base para crear variantes

const BotonEditar = styled(BotonAccion)`
border-color: #ffc107;
color: #ffc107;
&:hover {
background-color: #ffc107;
color: white;
}
`;
const BotonEliminar = styled(BotonAccion)`
border-color: #dc3545;
color: #dc3545;
&:hover {
background-color: #dc3545;
color: white;
}
`;
// ... (lógica del componente Gestion)
<BotonEliminar onClick={() => handleDelete(prod.id)}>Eliminar</BotonEliminar >
<BotonEditar onClick={() => handleEditClick(prod)}>Editar</BotonEditar>

Explicación del código:
BotonAccion → es un <button> normal, pero con estilos encapsulados que solo afectan a este componente.
styled(ComponenteBase) → nos permite heredar estilos de otro componente y luego agregar o sobrescribir reglas.
BotonEditar hereda de BotonAccion y cambia colores al estilo “warning”.
BotonEliminar hace lo mismo, pero con estilo “danger”.
Reutilización de código (principio DRY):

En lugar de repetir CSS en cada botón, centralizamos la base en BotonAccion y solo modificamos lo necesario en cada variante.
Encapsulamiento: los estilos solo aplican dentro del componente y no “ensucian” el CSS global.

Ventajas de styled-components
✅ Estilos acoplados al componente (fáciles de mantener).
✅ Reutilización: podemos crear un estilo base y extenderlo en varias variantes.
✅ Uso de pseudoclases (:hover, :focus) de manera natural dentro del componente.
✅ Legibilidad: en el código vemos directamente cómo luce el componente, sin tener que saltar a un archivo CSS.

React Icons
react-icons nos permite importar íconos SVG como si fueran componentes de React.
Simplemente los colocamos dentro de nuestro JSX. Se comportan como cualquier otro elemento, por lo que podemos aplicarles 
estilos en línea o a través de clases para ajustar su apariencia y espaciado.

Instalación de Dependencias:

npm install react-icons

Ahora, agreguemos íconos a nuestros botones de acción en Gestion.jsx.
// En /src/componentes/Gestion/Gestion.jsx
import { FaEdit, FaTrash } from 'react-icons/fa'; // Importamos los íconos
como componentes
// ...dentro del return, en los botones estilizados
<BotonEditar onClick={() => manejarEditar(prod)}>
<FaEdit style={{ marginRight: '5px' }} /> Editar
</BotonEditar>
<BotonEliminar onClick={() => handleDelete(prod.id)}>
<FaTrash style={{ marginRight: '5px' }} /> Eliminar
</BotonEliminar>

Estilización
Al ser componentes, podemos darles estilos de distintas maneras:
Con clases o CSS externo
{/* En el return del componente */}
<FaEdit className="icono-editar" />
/* En su css correspondiente */
.icono-editar {
color: #ffc107;
margin-right: 5px;
font-size: 20px;
}
Con estilos en línea
{/* Directamente en el return */}
<FaTrash style={{ color: "red", fontSize: "18px" }} />


Optimización para Buscadores (SEO) con React Helmet

React Helmet es una librería que te permite controlar el <head> de tu documento HTML directamente desde tus componentes de React.
Te deja escribir etiquetas como <title>, <meta> o <link> dentro de cualquier componente. React Helmet se encarga de tomar esas
etiquetas y moverlas al <head> del DOM real en el navegador.
Su principal utilidad es mejorar el SEO (Optimización para Buscadores). Las aplicaciones de React (SPAs) normalmente tienen 
un único <head> estático, lo que es malo para que Google entienda tus distintas páginas. Con React Helmet, cada ruta 
(/producto/1, /contacto, etc.) puede tener su propio título y descripción, haciendo que cada página sea única y legible para 
los buscadores.

Instalación de Dependencias:

npm install react-helmet

Lo aplicamos en la página de detalle del producto para tener títulos dinámicos.
// En src/componentes/ProductosNacionales/ProductosNacionalesDetalle.jsx
import { Helmet } from 'react-helmet';

// En src/componentes/ProductosNacionales/ProductosNacionalesDetalle.jsx
return (
{prod && (
<Helmet>
<title>Mi Tienda | {prod.nombre}</title>
<meta name="description" content={`Detalles y precio del
producto ${prod.nombre}.`} />
</Helmet>
)}
<div className="container">
{/* Características del producto */}
) : (
<p>Cargando producto...</p>
)}
</div>
</div >
);

Explicación del código:
El componente <Helmet> actúa como un contenedor para las etiquetas que deseamos inyectar en el <head> del HTML. React Helmet 
se encarga de extraer estas etiquetas y actualizar el DOM correspondiente. Esto es crucial para el SEO, ya que permite que 
cada página tenga un título y una descripción únicos que los motores de búsqueda pueden indexar.
Al mostrar el detalle de algún producto, puedo ver en el inspector del navegador, que el contenido de la descripción es 
dinámico. Te invito a que lo pruebes con otros productos.
