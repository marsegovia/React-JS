Flujo de datos: De contenedor a presentacional
Ahora vamos a aplicar el patrón que ya conocemos, pero en una estructura de tres niveles, algo súper común en 
aplicaciones reales. Queremos mostrar una lista de productos en nuestra tienda. Para lograrlo de forma ordenada y 
escalable, vamos a repartir el trabajo en tres componentes especializados, cada uno con una misión clara: Dentro de 
nuestra carpeta Componentes, vamos a crear los componentes que vemos entre paréntesis.

1. El Cerebro (ItemListContainer): Es el componente contenedor. Su única misión es tener la lógica y conseguir los 
datos. No le preocupa en lo más mínimo cómo se van a ver esos datos.
2. El Organizador (ItemList): Este componente es un intermediario. Recibe la lista completa de datos desde 
"El Cerebro" y su trabajo es organizar y preparar esa información. Se encarga de recorrer la lista y delegar la 
tarea de mostrar cada ítem individual.
3. El Exhibidor (Item): Este es el componente presentacional puro. Es el encargado de la apariencia. No sabe de 
dónde vienen los datos, ni le importa. Solo recibe la información de un único producto y se concentra en mostrarlo 
de la forma más prolija y atractiva posible.

Nivel 1: El Cerebro - ItemListContainer.jsx
Este es nuestro punto de partida. Como componente contenedor, su responsabilidad es manejar los datos. En una 
aplicación real, acá haríamos una llamada a una base de datos o una API para buscar los productos. Para nuestra 
clase, vamos a simularlo creando un array directamente en el código.

    // En /componentes/ItemListContainer/ItemListContainer.jsx
    
        import { ItemList } from "../ItemList/Itemlist";
        export function ItemListContainer({ Mensaje }) {
            const productos = [
                { id: '1234', nombre: 'Notebook Pro', precio: 12000, stock: 15 },
                { id: '2344', nombre: 'Monitor Curvo', precio: 450000, stock: 25 },
                { id: '2545', nombre: 'Teclado Mecánico', precio: 15000, stock: 50 },];
            
            return (
                <div>
                    <h2>{Mensaje}</h2>
                        <div>
                            <ItemList productos={productos} />
                        </div>
                </div>
            );}

En resumen: ItemListContainer imprime un mensaje en h2 y obtiene los datos que luego se los pasa a su hijo, 
ItemList. No hace más.


Nivel 2: El organizador - ItemList.jsx
Este componente recibe la lista. No le importa cómo se ve cada producto, solo se encarga de recorrer el array 
(.map()) y decirle al siguiente nivel que renderice cada uno. ItemList recibe una lista y la transforma en múltiples
componentes Item, pasándole a cada uno los datos que le corresponden.

    // En /components/ItemList/ItemList.jsx
        import { Item } from "../Item/Item";
        export function ItemList({ productos }) {
            return (
                <div style={{ display: 'flex', gap: '20px' }}>
                    {productos.map(prod => (
                    <Item key={prod.id} {...prod} />
                    ))}
                </div>
            );
    }

Detalles del código:
1. Con el div agrupamos los elementos y le damos estilo en línea a este grupo puntualmente.
2. Con el mapeo, lo que estamos haciendo es lo siguiente: para cada "prod" en el array "productos", creamos un 
componente "Item", al que le pasamos una "key" única. (Muy importante para optimizar)
3. Usamos el spread operator {...prod} para pasar todas las propiedades del objeto "prod" (id, nombre, precio, stock)
como props individuales al componente Item 

Nivel 3: El exhibidor - Item.jsx
Este es nuestro componente presentacional o "dumb". No sabe de listas ni de lógica. Solo recibe las props de un
producto y se encarga de mostrarlo de la forma más linda posible, usando sus propios estilos de CSS Modules.

    // En /componentes/Item/Item.jsx
    // Recibe las props usando destructuring
        export function Item({ nombre, precio, stock }) {
            return (
                <div>
                    <h3>{nombre}</h3>
                    <p>Precio: ${precio}</p>
                    <p>Stock disponible: {stock}</p>
                    <button>Comprar</button>
                </div>
            );
        }

Como quedaría entonces el flujo de nuestro código: Container tiene los datos -> se los pasa a List -> List mapea y 
le pasa los datos de cada uno a Item -> Item los muestra. ¡Perfecto!

Paso final: El ensamblaje en App.jsx y diseñar

Ensamblado
Ya tenemos toda nuestra cadena de componentes lista. Ahora, para que todo esto se vea en nuestra página, 
necesitamos "llamar" al componente de principal de nuestra cadena, el ItemListContainer, desde nuestro componente 
principal, que es App.jsx. App.jsx es el centro de nuestra aplicación. Desde acá orquestamos qué componentes se
muestran en la página.

// /src/App.jsx
import './App.css';
import { Layout } from './componentes/layout/Layout';
import { ItemListContainer } from
'./componentes/ItemListContainer/ItemListContainer';

function App() {
return (
<Layout>
{/* Todo lo que pongamos acá adentro irá donde estaba {children} */}
<h1>¡Bienvenidos a mi página!</h1>
<p>Este es el contenido principal.</p>
<ItemListContainer Mensaje="Nuestros productos destacados" />
</Layout>
);
}
export default App;

Diseño
Para que nuestro componente quede mas lindo, vamos a aplicarle CSS modules. Para eso, nos vamos a nuestra carpeta 
del componente, y creamos el modulo par ese componente. Recordá que es el mismo nombre que el archivo .jsx con la 
extensión module.css.

Voy a usar flex para centrar los productos y padding para distanciarlos
/* En /componentes/ItemListContainer/ItemListContainer.module.css*/
.subtitulo{
padding:1rem 1rem 0 1rem;
}
.productos{
display: flex;
flex-direction: row;
justify-content: center;
align-items: center;
padding:2rem;
}
