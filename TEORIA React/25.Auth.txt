Autenticación vs. Autorización:

- Autenticación: Es el proceso de verificar la identidad de un usuario. Responde a la pregunta: "¿Quién sos?".
Cuando un usuario inicia sesión con su correo y contraseña, lo estamos autenticando.
- Autorización: Es el proceso de definir los permisos que tiene un usuario una vez autenticado. Responde a la pregunta: 
"¿Qué tenés permitido hacer?". Solo los administradores puedan gestionar productos.

Autenticación con Firebase
Firebase nos ofrece un servicio robusto y muy simple de implementar para toda la gestión de autenticación de usuarios, 
para eso necesitamos preparar nuestro proyecto de Firebase para que acepte autenticación. Lo haremos siguiendo estos pasos:
1. Habilitar el proveedor de autenticación: En la consola de su proyecto, navega a la sección Compilación > Authentication.
Hacé clic en "Comenzar" y seleccionen el proveedor "Correo electrónico y contraseña". Simplemente lo activas y 
guardas los cambios.
2. Crear un usuario de prueba: Para facilitar el desarrollo, crearemos un usuario manualmente. En la misma sección de 
Authentication, vayan a la pestaña "Users" y hagan clic en "Add user". Pueden utilizar credenciales fáciles de recordar, por
ejemplo:
○ user: admin@gmail.com
○ password: admin1234

Con esto, nuestra plataforma ya está lista para autenticar.

Creamos el componente de Login
En el código crearemos un formulario super sencillo en un nuevo componente para que el usuario pueda ingresar sus credenciales.
Análisis del Código:
● getAuth(): Esta función nos devuelve la instancia del servicio de autenticación de Firebase asociado a nuestro proyecto.
● signInWithEmailAndPassword(auth, email, password): Es la función principal que ejecuta la lógica de inicio de sesión. 
Es asincrónica y devuelve una Promesa.
● Manejo de la Promesa:
    ○ Si las credenciales son válidas, se ejecuta el bloque .then(), dándonos acceso a la información del usuario que ha 
    iniciado sesión.
    ○ Si ocurre un error (usuario inexistente, contraseña incorrecta, etc.), se ejecuta el bloque .catch() para que podamos 
    gestionar el error adecuadamente.

    // src/componentes/Login/Login.jsx
    import React, { useState } from 'react';
    import { getAuth, signInWithEmailAndPassword } from "firebase/auth";
    import { useNavigate } from 'react-router-dom';
    
    const Login = () => {
    const [email, setEmail] = useState("");
    const [password, setPassword] = useState("");
    const navigate = useNavigate();
    const handleLogin = (e) => { 
        e.preventDefault();
        const auth = getAuth();
        signInWithEmailAndPassword(auth, email, password).then((userCredential) => {
            const user = userCredential.user;
            console.log("Usuario logueado:", user);
            alert("¡Inicio de sesión exitoso!");
            navigate('/'); //   
        }).catch((error) => {
            const errorCode = error.code;
            const errorMessage = error.message;
            console.error("Error en el login:", errorCode, errorMessage);
            alert("Error: " + errorMessage);
        });
    };

    return (
        <div>
            <h2>Iniciar Sesión</h2>
            <form onSubmit={handleLogin}>
                <input type="email" placeholder="Correo electrónico" value={email} onChange={(e) => setEmail(e.target.value)}/>
                <input type="password" placeholder="Contraseña" value={password} onChange={(e) => setPassword(e.target.value)}/>
                <button type="submit">Ingresar</button>
            </form>
        </div>
    );
};

export default Login;

Autorización: Implementando un sistema de roles
Firebase Authentication maneja la identidad, pero para datos adicionales como los roles, utilizaremos Cloud Firestore. 
Para eso, crearemos una colección en Firestore que vincule el UID de un usuario con su rol.

Primero vamos a crear una Colección usuarios en la consola de Firebase yendo a Firestore Database y creando una nueva 
colección llamada usuarios. Para nuestro usuario admin@gmail.com, crearemos un documento manualmente.
● ID del Documento: Debe ser el UID exacto del usuario administrador. (Puedes encontrar el UID en la pestaña Authentication 
de la consola de Firebase).
● Campo: Agregá un campo rol con el valor admin. Cualquier otro usuario que se registre no tendrá este documento y, 
por lo tanto, lo trataremos como un usuario con rol estándar.

Un estado de autenticación global con Context API
Nuestro componente Login ya funciona, pero la aplicación no "sabe" si un usuario está autenticado o no. No tiene sentido 
mostrar un botón de "Login" a un usuario que ya ha iniciado sesión.
Para que cualquier componente pueda acceder al estado de autenticación (quién es el usuario, o si hay un usuario), 
necesitamos un estado global. La solución, como ya hemos visto para el carrito de compras, es la Context API.

Crearemos un contexto específico para la autenticación.
    // En /context/AuthContext.jsx
    import React, { createContext, useState, useContext, useEffect } from 'react';
    import {getAuth,onAuthStateChanged,signOut,createUserWithEmailAndPassword,signInWithEmailAndPassword} from "firebase/auth";
    
    // 1. Crear el contexto
    export const AuthContext = createContext();
    // Hook personalizado
    export const useAuth = () => {
        const context = useContext(AuthContext);
        if (!context) {
        throw new Error("useAuth debe ser usado dentro de un AuthProvider");    
        }
    return context;
    };
    
    // 2. Crear el proveedor del contexto
    export const AuthProvider = ({ children }) => {
        const [user, setUser] = useState(null);
        const [loading, setLoading] = useState(true);
        const auth = getAuth(); // Obtenemos la instancia de auth una sola vez
        const db = getFirestore(); // Inicializamos Firestore
        
        // Función para registrar un nuevo usuario
        const signup = (email, password) => {
            return createUserWithEmailAndPassword(auth, email, password);
        };
    
    // Función para iniciar sesión
    const login = (email, password) => {
        return signInWithEmailAndPassword(auth, email, password);
    };
    
    // Función para cerrar sesión
    const logout = () => {
        signOut(auth);
    };
    
    useEffect(() => {
    // onAuthStateChanged es el observador de Firebase
        const unsubscribe = onAuthStateChanged(auth, async (currentUser) => {
            if (currentUser) {
            // Si hay un usuario, buscamos su rol en Firestore.
            const userDocRef = doc(db, "usuarios", currentUser.uid);
            const userDocSnap = await getDoc(userDocRef);
            if (userDocSnap.exists() && userDocSnap.data().rol ==='admin') {
                // Si el documento existe y tiene rol de admin, lo asignamos.
                setUser({ ...currentUser, rol: 'admin' });
            } else {
                // Para cualquier otro caso, es un usuario regular.
                setUser({ ...currentUser, rol: 'user' });
            }
        } else {
            setUser(null);
        }
    setLoading(false);
    });
    // Limpiamos el observador al desmontar
    return () => unsubscribe();
    }, [auth, db]); // Agregamos 'auth' como dependencia
    
    // Crear el objeto 'value' con TODAS las funciones definidas
    const value = {
        user, loading, // Es buena práctica pasar el estado de carga también
        signup, login, // Ahora 'login' sí existe y se puede pasar
        logout,
    };
    
    // Retornar el Provider, asegurándonos de no renderizar hasta que cargue
    // Esto evita que los componentes hijos puedan acceder a 'user' cuando es null
    return (
        <AuthContext.Provider value={value}>
            {!loading && children}
        </AuthContext.Provider>
    );
};

Explicación del código:
AuthContext: Crea un espacio compartido donde la información del usuario estará disponible para toda la aplicación.
AuthProvider: Este componente hace el trabajo pesado:
● Mantiene el estado: Guarda en su memoria interna quién es el user (el usuario actual) y si la información aún está loading
● Provee las herramientas: Ofrece las funciones para que otros componentes puedan login, logout y signup sin tener que 
hablar directamente con Firebase.
● Escucha a firebase: Usa useEffect con onAuthStateChanged para poner un "oyente" que le avisa automáticamente a la 
aplicación cada vez que el estado de autenticación cambia (alguien inicia o cierra sesión).
useAuth: Es la llave de acceso 🔑. Es un atajo para que cualquier componente pueda pedirle al AuthProvider la información 
del usuario y las herramientas (login, logout, etc.) de forma fácil y segura.
