CRUD

CRUD es un acrónimo que representa las cuatro operaciones fundamentales para la gestión de datos persistentes en cualquier 
sistema de software. Estas operaciones son el pilar de la mayoría de las aplicaciones que interactúan con una base de datos.

● C - Create (Crear): La capacidad de agregar nuevos registros o entradas de datos. En nuestro caso, dar de alta un nuevo producto.
● R - Read (Leer): La habilidad de consultar, obtener y mostrar los datos existentes.
● U - Update (Actualizar): La funcionalidad para modificar o editar registros que ya existen. 
Por ejemplo, cambiar el precio o el stock de un producto.
● D - Delete (Eliminar): La operación que nos permite borrar permanentemente un registro de la base de datos.
Dominar el ciclo CRUD es dominar la interacción fundamental entre una aplicación y sus datos.

¿Cómo funciona en React?
Es crucial entender que React, como librería de front, no realiza operaciones CRUD por sí misma. React se encarga de construir 
la interfaz de usuario (formularios, botones, listas) que permite al usuario iniciar estas operaciones.
El flujo de trabajo es el siguiente:
1. Vista (React): Un usuario interactúa con un componente, por ejemplo, llenando un formulario y presionando "Guardar".
2. Evento: Esta interacción dispara un evento (onSubmit) que ejecuta una función en nuestro código.
3. Lógica (JavaScript + Firebase): La función contiene la lógica para comunicarse con el backend (Firebase). 
Llama a la función correspondiente de Firestore (addDoc, updateDoc, deleteDoc).
4. Actualización de Estado: Una vez que Firebase confirma que la operación fue exitosa, actualizamos el estado local de nuestra 
aplicación con useState para que los cambios se reflejen en la interfaz de inmediato, sin necesidad de recargar la página.

Comencemos con una estructura básica que simplemente lista los productos.

Eliminar producto (Delete)
La eliminación de un documento en Firestore es una operación directa. La librería nos provee de la función deleteDoc.

deleteDoc de Firestore:
Para eliminar un documento, necesitamos dos cosas:
● La referencia exacta a ese documento.
● Llamar a deleteDoc pasándole dicha referencia.
La referencia se crea igual que cuando buscamos un único documento, utilizando la función doc.

Lo que haremos:
Haremos la lógica para eliminar un producto. Importaremos la función deleteDoc de Firebase y crearemos un manejador de eventos 
handleDelete. Esta función crea una referencia al documento específico por su id y lo elimina de la base de datos.
Inmediatamente después, actualiza el estado local de React para que el producto desaparezca de la lista en la pantalla.

Explicación detallada
1. Importaciones:
○ Además de collection y getDocs que ya utilizábamos, ahora importamos dos nuevas funciones:
■ doc: Usada anteriormente. Crea la referencia.
■ deleteDoc: Es la función que ejecuta la acción de borrado sobre la referencia que le pasemos.

2. Función handleDelete:
○ Esta función asíncrona recibe como parámetro el id del producto que el usuario desea eliminar. Este id lo obtendremos del

3. Confirmación del usuario:
○ La eliminación es una acción destructiva e irreversible. Es una excelente práctica de experiencia de usuario (UX) pedir 
siempre una confirmación antes de proceder. window.confirm muestra un diálogo nativo del navegador y devuelve true si el 
usuario acepta.

4. Lógica de eliminación:
○ Todo el proceso de borrado se ejecuta únicamente si el usuario confirmó la acción.
○ const docRef = doc(db, "Productos nacionales", id);: Creamos la referencia exacta al documento que queremos borrar.
○ await deleteDoc(docRef);: Ejecutamos la eliminación en Firebase y, gracias a await, esperamos a que la operación 
se complete antes de continuar.

5. Actualización de la interfaz (UI):
○ Para que el usuario vea el cambio reflejado inmediatamente, actualizamos nuestro estado productos. Utilizamos el método 
.filter() para generar un nuevo array que contiene todos los productos excepto aquel cuyo id coincide con el que acabamos de 
eliminar. React detecta este cambio de estado y vuelve a renderizar la lista, ya sin el ítem borrado.
